What the pills mean

Every badge you'll see on a worklist card or address row, in plain language.

Misroute — scanned Area NThis package is inbounding to the wrong work area for its true (corrected) address — pull it before it loads. Named area(s) are where it actually scanned; the "Right now" line and each misrouted chip's "→ Area N" annotation say the same thing in different places.
Area N · 9 of 11The corrected address's Mainly Assigned Area (the dominant area), learned from the Manual Assignment Detail report. Where the assignment history is not unanimous, the split shows too ("9 of 11") so a thin majority reads as evidence, never gospel — a bare "Area N" means every sighting agreed.
Both reportsThis correction was seen in both the Address Corrections Report and the Manual Assignment Detail report — the strongest trust signal.
Delivery-confirmed ✓At least one past package sent to this corrected address was confirmed DELIVERED there (the Address Corrections Report's Delivered Address column agreed with the corrected address) — proof the correction demonstrably worked, not just that it was applied. A stronger trust signal than the plain "seen N×" count.
Recurring IBNOAt least one package at this address is also a Recurring IBNO — the bad address is why it keeps coming back.
seen N×How many times this correction has been seen across the loaded Address Corrections / Manual Assignment reports — the evidence behind the suggestion.
Coded 02/33QA already scanned this package 02 (bad address) or 33 (address search) — it was already touched for an address reason. Its ABSENCE on a match is the point: it means ITQA/QA has not caught this one yet.
N on truckHow many packages at this address have already been van-scanned (loaded). Splits "still catchable at the belt" from "go pull it off the truck."
VerifyName match — verify: a past correction added a unit/suite to this address for a specific recipient — normally suppressed as person-specific, not a standing address fix. It only reappears here because today's package's label name matches that recipient's name closely. This is a LEAD, not a trusted fix — even a matching name can legitimately route to the office instead of the unit. Always verify before acting.
Evidenced historyWrong area — correct address: which Misroute Trigger track fired. "Evidenced history" = the mainly-assigned area is dominant (≥80%, ≥3 sightings). "Mass divergence today" = the history is thin but consistent AND ≥2 packages went to the SAME wrong area today — the mass-missort signature. Today's scans agreeing with each other is NOT proof they're right; the manual-assignment record is the authority.
Suggest Area N · 1 of 1A package scanned 999 at an address whose mainly-assigned area is already known from the manual-assignment history — a head start on the FRO/DRO lookup, with its tally as evidence. A suggestion to verify, never a misroute flag.

Load an Address Corrections Report to learn standing corrections into the Address Correction Dictionary — and a Manual Assignment Detail at IB Scan report to corroborate those mappings, learn each corrected address's proper work area, and build the standalone Area Dictionary (address → mainly-assigned area, checked against every inbound package) — then drop today's live Inbound and Van Scans report to see which incoming tracking numbers need a known correction, which correct-address packages are heading to the wrong area, and a suggested area for today's 999s at known addresses.
